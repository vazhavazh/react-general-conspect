SPA(single-page-application)

A single-page application is an app that works inside a browser and does not require page reloading during use. You are using this type of applications every day. These are, for instance: Gmail, Google Maps, Facebook or GitHub. // So client request only one per pages HTML, JS, CSS. // for example: If client want to get to another page by navigation, // this is executing on front end // no need more request to server( back-end) // also client still have oportunity to make request to back-end and ask for some data

SPA advantages:

1) Web page is not reloading during navigation over the page 2) Instantly(МОМЕНТАЛЬНО) re-rendering web-page' s elements, cause we have ability rerender certain(ОПРЕДЕЛЁННЫЕ) elements(only which we want), not all page 3) Easier for publication in production cause it have small quantity of static files 4)Internet connections will be more effective due to decrease their quantity and decrease delay between requests and only first acess to site will be "long" and other one "react"

SPA disadvantages:

1) For SEO need special options, becuse SPA have only one page 2) Long first connection due to big JS file

REACT BASE

Reloading whole DOM is hard operation. React have 'virtual DOM' which is only JavaScript's object So before changing real DOM react change virtual DOM and as virtual DOM is object, it happens almost instantly. So per each change on page firstly change virtual DOM only after real DOM Example: Botton clicked // so state of some element(button) has changed react firstly change state in the element which is in virtual DOM but as this element has children, then changing his state will call changing chuldren's state automatically. And this is give ability also to change them or to left them // After react compare 2 DOM's states(VERSIONS or CONDITIONS): CURRENT AND PREVIOUS React compares it by each changed element by comparing their state(СОСТОЯНИЯ) to find what exactly change in element. After react defines which exactly elemnts were changed and which with out changes(differences): (for example one element changed his 2 children also chenged another 3 children were left with out changes) So only this changed lements wil be re-rendered in real DOM. There is no full page re-render.There is only selective rendering(Точечная перересовка)ю So Virtual DOM collected little bit changes and then only real DOM rendering

BASE CONCEPTS IN REACT

-Components (building blocks of React applications. They are independent and reusable parts of the user interface that can have their own internal state and properties (props).) - JSX (Jsvascript Syntax Extencion) This is a JavaScript syntax extension that allows combining HTML-like(CSS) tags and JavaScript expressions into a single React component.
